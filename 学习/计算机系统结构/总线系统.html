总线系统

简介
...

处理器通过总线控制和传送数据

总线按功能分有：

  • 数据线
  • 地址线
  • 命令、时序、中断信号等控制/状态线
  • 电源线
  • 地线和备用线

总线的分类
...

按在系统中的位置分为：

  • 芯片级：CPU 芯片内的总线
  • 板级：连接插件板
  • 系统级：系统间或主机于 IO 接口或者设备之间的总线

按信息传输方向分为：

  • 单向传输
  • 双向传输
    • 半双向
    • 全双向：能同时双向

按用法分：

  • 专用
    • 缺点：总线数多，如果用 N 个部件双向专用总线需要 条
    • 优点：方便简单，
  • 非专用
    • 同一时间只有一对设备能用
    • 优点：总线数少，造价低，总线接口标准化，模块性强，可扩充能力强，部件增加不会使电缆、接口、驱动电路增加
    • 缺点：系统流量小，经常出现总线占用的情况，如果处理不当有可能称为系统性能的瓶颈

总线的控制方式
...

如果同时有多个设备请求使用总线，就得由总线控制机构决定优先次序裁决，保证只有一个高优先级的申请者首先取得对总线的使用权

总线控制机构有两种分布方式：

  • 集中式控制
  • 分布式控制

集中式控制
...

优先次序的确定有三种方式

串行连接
...

  • 有三条总线连接总线控制器，"总线可用" （串行），“总线请求”，“总线忙”
  • 所有部件经过公共的总线请求线向总线控制器发出要求使用总线的申请
  • 只有当总线忙信号未建立时，送出总线可用信号
  • 如果某个部件收到总线可用信号，但没有发出总线请求时，将信号发给下一个部件
  • 如果该部件收到总线可用信号，且发出过总线请求，则建立总线忙信号，并去除总线请求
  • 使用好总线后，去除总线忙信号

离总线控制器越近的部件优先级越高 优点：

  • 选择算法简单
  • 总线控制分配的控制线少，只需要 3 根，且不属于部件的数量
  • 部件的增减容易，只需要简单地把它连到总线上 缺点：
  • 对“总线可用”线及其有关电路的失效敏感，如果有一个部件失效，那整体失效
  • 优先级由线固定，不能由程序改变，不灵活
  • 如果高优先级的部件连续使用总线，则低优先级的部件很难使用总线
  • 总线可用信号按顺序流过每个部件，限制了总线的分配速度
  • 受总线长度限制

定时查询
...

  • 每个部件通过总线请求线发出请求
  • 若总线忙未建立，则总线控制器收到请求后开始计数，定时查询各部件确定是谁发的请求
  • 查询线上的计数值与部件号一致时，该部件建立总线忙，计数器停止计数
  • 去除该部件的总线请求，让该部件获得总线使用权
  • 传送完数据后，去除总线忙

如果每次使用完，计数器清 0，则按照优先级分配使用权 若不清 0，则是一种循环优先级，为每个部件提供相同的使用权

优点：

  • 计数器初值、部件号均可由程序指定，优先级可以用程序控制，灵活性强
  • 不会因为某个部件失效导致其他部件对总线的使用，可靠性高

缺点：

  • 使用总线较多，需要 根总线
  • 可以共享总线的部件数受限于定时查询线的线数，扩展性较差
  • 控制较为复杂
  • 总线分配的速度取决于计数信号的频率和部件数，速度不会很快

独立请求
...

  • 每个部件各自有一对总线请求和总线准许线
  • 当部件请求使用总线时，送总线请求信号到总线控制器
  • 如果总线闲着，总线控制器根据某种算法对同时送来的多个请求进行仲裁，确定哪个部件使用总线
  • 确定使用总线的部件后，通过相应的总线准许线送会该部件，去除其请求，建立总线已被分配
  • 该部件获得总线控制权，使用完总线后，部件去除总线已被分配，经总线控制器去除总线准许

优点：

  • 总线分配速度快，所有部件总线请求同时送到总线控制器，不需要查询
  • 控制器可以使用程序可控的预定方式，自适应方式，循环方式，或它们的混合方式灵活确定下个使用总线的部件
  • 能方便地隔离失效部件的请求

缺点：

  • 控制线数量给过大，为控制 N 个设备需要 2N+1 条控制线
  • 总线控制器复杂

总结
...

小、微型机上主要采用串行连接方式，因为简单便于实现 后两者大多使用在巨型、大型机器

总线通信技术
...

同步通信
...

部件间的信息传送由定宽、定距的系统时标同步：

  • 优点
    • 信息传送速率高
    • 受总线长度影响小
  • 缺点
    • 时钟在总线上的时滞可能导致同步误差
    • 时钟线上的干扰信号容易引起误同步

异步通信
...

异步单向源控制
...

  • 源部件将数据放在数据总线上，延迟 后在控制总线上发送数据准备信号
  • 是为了防止数据准备信号先于数据到达，是源部件输出寄存器再装入新的数据或总线重新分配的时间

异步单向目控制
...

  • 目的部件建立数据请求使源部件把数据放到数据线上
  • 目的部件发出数据请求，经过二次传送延迟 后，数据到达目的部件
  • 目的部件检验数据有效性，如果出错，目的部件回送数据出错代替下个数据请求

单向控制缺点
...

无法保证下个数据传输之前让所有数据线和控制线的点平信号恢复到初始状态，可能造成错误

非互锁方式异步双向控制
...

  • 源将数据放在总线上，经过 延迟发出数据准备
  • 目的部件在 时刻接收数据后校验未出错就发送数据接收来响应
    • 源部件在 时刻去除原数据
    • 经过 时间把新数据放在数据总线上
  • 如果发现有错，就发送数据出错代替数据接收

提供了数据出错控制，便于实现不同速度的部件之间的通信，代价是降低传输速率 因为要经过二次总线延迟，并添加某些控制逻辑

互锁方式异步双向控制
...

异步双向互锁方式虽然增加了信号沿总线来回传送的次数，使控制硬件变得复杂，但能适应不同速度的 IO 设备，保证数据的正确性，且有较高的数据传送速率

所以 IO 总线中应用最广泛的就是异步双向互锁方式
