指令系统设计和优化

指令系统设计基本原则
...

  • 规整性
  • 对称性
    • 在指令系统中的所有寄存器和存储器单元都可同等对待，所有指令都可使用各种寻址方式
  • 独立性和全能型
    • 同个功能的指令只有一个，且指令功能齐全
  • 正交性
    • 各个指令中的字段之间没有关系，更改 A 不会更改 B
  • 可组合性
    • 任意寻址方式和任意指令都可以组合
  • 可扩充性
  • 额外
    • 指令码密度适中
    • 兼容性
      • 向后兼容
    • 适应性

哈夫曼编码
...

当各种事件发生的概率不均等是，利用哈夫曼编码技术，对发生概率高的事件采用短的位数

指令集操作分类
...

  • 算数和逻辑运算：加减与或
  • 数据传输：Load/Store
  • 控制：分支、跳转、过程调用和返回、自陷
  • 系统：操作系统调用、虚拟存储器管理
  • 浮点：加/乘
  • 十进制：十进制加/乘、十进制到字符的转换
  • 字符串：字符串移动/比较/搜索
  • 图形：像素操作、压缩/解压缩

控制指令
...

  • 跳转：无条件改变控制流指令
  • 分支：有条件改变控制流指令

控制流改变情况
...

  • 分支
    • 使用频率最高
  • 跳转 jump
  • 过程调用 call
  • 过程返回 return

分支条件指令表示
...

由于分支条件使用频率很高，对于该类指令需要考虑进行优化

通过三种方式表示条件分支指令

  • 条件码（CC）：在程序控制下，由 ALU 操作设置特殊的位
    • 优点：可以自由设置分支条件
    • 缺点：必须从一条指令将分支条件传送到分支指令，所以 CC 是额外状态，条件码限制了指令执行顺序
  • 条件寄存器：根据比较结果测试条件寄存器
    • 优点：简单
    • 缺点：占用了一个寄存器
  • 比较分支：比较操作是分支指令的一部分、比较受限
    • 优点：一条指令完成了两条指令的功能
    • 缺点：分支指令操作增多

过程调用和返回状态保存
...

"调用者保存"
...

一个调用者调用别的过程时，必须保存自己所用到的寄存器，以备调用结束返回后，能再次访问调用者

"被调用者保存"
...

被调用的过程必须保存它用到的寄存器，保证不会破坏过程调用者的程序执行环境，并在过程调用结束返回后，恢复这些寄存器

总结
...

被调用者保存更为灵活，能够支持调用嵌套

所以现在往往使用被调用者保存的机制

指令系统的发展和改进
...

  • 一个方向是强化指令功能，实现软件功能向硬件的转移

  • 这类系统称为复杂指令集计算机（CISC）

  • 一个方向是精简指令集

  • 这类系统成为精简指令集计算机（RISC）

CISC 指令集功能设计
...

  • 面向目标程序增强指令功能
    • 提高运算型指令功能
    • 提高传送指令功能
    • 增加指令控制指令功能
  • 面向高级语言和编译程序进行改进
    • 增加对高级语言和编译系统的指令功能
    • 针对高级语言设计指令集系统
  • 面向操作系统的优化实现改进指令系统
    • 对中断处理、进程管理、存储管理和保护、系统工作状态的建立与切换等的支持
    • 如虚拟化（VT）相关的操作的指令支持
    • 可以设置系统工作状态和访问方式转移的指令、支持进程转移的指令、支持进程同步和互斥的指令等措施，达到优化操作系统的目的

CISC 结构存在以下缺点：

  • 各种类型指令使用频率相差悬殊
    • 20%的指令使用频率最大，占运行时间的 80%
  • 复杂性给 VLSI 设计增加负担，不利于单片集成
  • 许多复杂指令需要复杂操作，因而运行速度慢
  • 功能的不均衡性，不利于采用先进的计算机体系结构技术，如流水线技术

RISC 指令集功能设计
...

  • 选取使用频率最高的指令，并补充一些最有用的指令
  • 每套指令功能尽可能简单，并在一个机器周期完成
  • 所有指令长度相同
  • 只有 load 和 store 操作指令才能访问存储器，其他操作均在寄存器间进行
  • 以简单高效的方式支持高级语言
