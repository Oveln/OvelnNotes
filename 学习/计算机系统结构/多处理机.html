多处理机

简介
...

多处理机指的是有两台以上处理机，共享 IO 子系统，机间通过共享主存或高速通信网络通信

在统一操作系统控制下，协同求解大而复杂的问题的计算机系统

多处理机属于 MIMD 系统，和 SIMD 的阵列处理机相比，有很大区别，主要来源于并行性的不同：

  • 阵列处理机针对向量、数组处理，实现向量指令操作级的并行，属于开发中的同时性
  • 多处理机实现的是作业或任务间的并行，是开发中的并发性

在硬件结构上，多处理机要用多个指令部件分别控制，通过共享主存或机间互连网络实现异步通信 在算法上，不限于向量、数组处理，还要挖掘更多通用算法中隐含的并行性 在系统管理上，更依赖操作系统等软件手段，有效解决资源分析和管理，任务分配、处理机调度，进程同步和通信

多处理机的结构、机间互连、并行算法、并行语言、编译、操作系统等都会直接影响系统的性能和效率

多处理机的硬件结构
...

紧耦合
...

通过共享主存实现处理机间通信，通信速率受限于主存频宽 处理机数受限于互联网络带宽和各处理机访主存冲突的概率

为了减少访主存冲突：

  • 主存采用模 m 多体交叉存取，m 越大，冲突概率越小
  • 各处理机可自带小容量局部存储器存放该处理机运行进程的核心代码和常用系统表格
  • 还可自带 Cache 减少访存次数

构型 1
...

系统由 p 台处理机，m 个存储器模块和 d 个 IO 通道组成 通过处理机——存储器互联网络（PMIN），IO——处理机互联网络（IOPIN）和中断信号互联网络（ISIN）进行互连

PMIN 实现各处理机和各存储器模块的连接，使之经过仲裁后，每个模块在一个周期内只响应一台处理机的访存请求，为了减少各个处理机同时访问统一存储器模块的冲突，存储器模块数 m 应该等于或略大于处理机数 p

构型 2
...

基本与构型 1 相同，不同点在于每台处理机自带一个Cache

每个处理机有自己的 MM 和 ULM 作为局部缓存：

  • 减少访主存信息量
  • 降低访主存冲突概率
  • 减少处理机——存储器互连网络的冲突
  • 再自带 Cache 就可以进一步减少冲突 处理机间通过中断信号完了过，由一台处理机向另一台处理机发出中断信号实现进程同步 处理机和连接外设的 IO 经 IO——处理机互连网络实现通信，但这样价格贵，所以多数处理机采用非对称互连，即连到一台处理机的设备不能被另一台处理机直接访问 为了防止某台处理机失效导致所接的外设无法连接到别的处理机，可以采用一定的冗余连接

分类
...

  • 同构对称型
    • 每个处理机地位相等
  • 异构非对称型
    • 主处理机不同于从处理机，主处理机经高速系统总线连接字符处理机和字符外设，数组处理机，数组外设，网络处理机，图形处理机，向量加速处理机，从处理机均为专门的，结构不同于主处理机

松耦合
...

每条处理机都有一个容量较大的局部存储器，用于存储经常用的指令和数据，以减少访主存冲突

不同的处理机间通过通道互连实现通信，共享外部设备

通过消息传送系统（MTS）交换信息，每台处理机都有自己的外部设备：

  • 消息传送系统常用
    • 分时总线
    • 环形
    • 星形
    • 树形
    • 等拓扑结构

松耦合处理机适合做粗粒度的并行计算：

  • 处理的作业分割成若干各相对独立的任务，在各个处理机上并行
  • 任务间流量少
  • 处理机间任务间交互作用少

这种任务，松耦合的处理系统是很有效的，可以看作是一个分布系统

非层次型
...

典型的经消息传送系统互联的松耦合非层次型多处理机：

  • 有 N 个计算机模块（结点）
    • 每个结点中有 CPU、Cache、局部存储器
    • 一组 IO 设备
    • 还有一个通道和仲裁开关（CAS）
      • CAS 的通道中有一个高速通信存储器缓冲传送的信息块，该通信存储器经过 MTS 可被所有处理机访问
    • 消息传送系统（MTS）接口
      • 用于两个或多个结点同时请求访问 MTS 的某个物理段时进行仲裁，按照一定算法允许其中一个请求和延迟其他请求

MTS 可以是单总线，所有通信存储器连在分时使用的单总线上，也可以是总线数较少的多总线 因为总线上传送的速度要求不高，互连网络的成本又比紧耦合第，因而可以构成数百甚至数千台微处理机组成的多处理机 MTS 也可以是共享的网络通信系统，在松耦合的多处理机中，不同处理机的通信都通过通信存储器进行，而同一处理机间的通信只需要经过局部存储器完成

层次型总线式多处理机
...

所有的计算机模块通过两级总线按层次连接，map 总线可连接多大 14 个处理机模块，组成一个计算机模块组，用来加强组内各处理机的协作

用较低的通信开销实现数据共享

连在 Map 总线的 是各计算机模块组间的连接器，为了提高可靠性，多个模块组之间通过两条组间总线，连接成一个完整的 外系统，用包交换方式通信

机间互连形式
...

机间互连形式是决定多处理机性能的一个重要因素 要求：

  • 高通信速率
  • 低成本
  • 互连灵活多样
    • 实现复杂的不规则互连而不冲突

一般形式：

  • 总线
  • 环形互连
  • 交叉开关
  • 多端口存储器
  • 蠕虫穿洞寻径网络

当处理机较多时，也有类似 SIMD 的多级完了过，不同的是，每个互连开关都有小容量存储器，用来缓冲信息 对采用分布结构的多处理机则采用开关枢纽结构形式

总线形式
...

  • 单总线 - 优点 - 结构简单 - 成本低 - 增减模块方便 - 缺点 - 对总线失效敏感 - 处理机数增加会增大总线冲突率 提高效率办法：
  • 用优质高频同轴电缆提高速率，进一步的使用光纤通信
  • 采用多总线方式减少总线冲突概率
    • 如 处理机就采用了分级多总线

为了采用同时访问公用总线的冲突：

  • 静态优先级
    • 给每个部件分配固定的优先级
  • 固定时间片
    • 把总线按固定大小的时间片轮流提供服务
  • 动态优先级
    • 部件的优先级按一定规则动态改变
      • 如近期最少使用 LRU：有限响应最长时间没有使用过总线的部件
      • 循环串行链 RDC：每个周期按"总线可用"线所接的顺序，有限响应刚使用过总线的部件位置最近的部件发出的访问请求
  • 先来先服务
    • 能使部件的平均等待时间最少，但需要有机构记录各个请求到达的先后次序
  • 等多种总线仲裁算法

环形互连形式
...

各台处理机之间点点相连成环状，称为环形互连 这种多处理机上，信息传递由发送进程发送到环上，经由网络不断向下一台处理机传递，直至回到发送者处为止

发送信息的处理机有一个唯一的令牌，是普通传送的信息中不会出现的特定标记，同一时间只能有一个处理机持有令牌，发送者发送消息时，其他处理机都处于等待接收消息的状态，一旦发送者发送完就向环上广播令牌，其他处理机一次传递消息和令牌，并根据需要接收消息

由于环形连接是点对点连接，物理参数容易控制，适合于高通信带宽的光纤通信

缺点：

  • 每个接口处都会有一个单位的传输延迟，当处理机数增加时，延迟增大
    • 但不会因为处理机数量多，带宽就下降

所以环形互连适用于单个处理机单次发送长数据的情景

交叉开关形式
...

交叉开关形式用纵横开关将横向的处理机和 IO 通道以及纵向的存储器模块连接起来 交叉开关形式是多总线朝总线数增加的方向发展的极端情况 总线数= 相连模块数, 每个设备都能分到总线和 m 个存储器模块之一连通并通信

互联网络不争用开关，可以扩展带宽，提高系统效率

交叉开关是空间分配机制，任何设备交换信息的时候，只在开关处有一个单位的延迟

当然，如果多个处理机或 IO 通道访问同一个主存模块或访问共享主存变量，还会发生冲突，但这是访存冲突，而不是网络冲突

同时，交叉开关还需要有处理机访问存储器模块冲突的仲裁硬件，加上总线有一定宽度，因此，整个交叉开关阵列式十分复杂的，若纵向横向的设备都是 n，则总设备量将会是 ，当 n 很大，成本可能会超过其 2n 个部件的成本，得不偿失

因此，交叉开关的多处理机一般 ，少数有

多级交叉开关
...

由于交叉开关复杂，可以用多个较小规模的交叉开关串连和并连，构成多级交叉开关网络，取代单级大规模交叉开关

榕树形互连网络（Delta 网）
...

如果互联网络的出端数和入端数不同，为了降低复杂性，可以采用榕树形互连网络，采用 的交叉开关模块 a 的入端和 b 的任一输出端相连，用 n 级 交叉开关模块就可以组成 的开关网络

这种网络与 SIMD 的互连网络类似，不同的是每个开关都有小容量缓冲存储器，使用信息包交换，而不是线路交换

多端口存储器形式
...

如果每个存储器模块都有多个访问端口，将分布在交叉开关矩阵中的控制、转移、优先级仲裁逻辑分别移到相应的存储器模块的接口中，就构成了多端口存储器的结构

多端口存储形式的中心是多端口存储器模块，多个存储器模块和端口连接：

  • 每个端口处理一个处理机 P 或 IO 通道的访问请求
  • 每个存储器模块按照对各个端口指定的优先级化解对他的访问冲突

这种形式的所有复杂性就转移到存储器模块中了，因此端口数量不能太多，且一经做定就不易改变，这对多处理机是不利的，处理机较少时，这种结构还是比较成功的

蠕虫穿洞寻径网络
...

在处理机间采用小容量缓冲存储器，实现消息分组的寻径存储转发

在这种网络中，消息分组又分成更小的组，同一个分组的小组以异步流水方式按序不间断传送，同一分组只有头部小组知道目的地，且用硬件方式进行传送的应答，各个分组允许交叉传送

但不同分组的各个小组不能混在一起传送，利用虚拟通道思想，使发送和接收结点间的一条物理通道，能被多个虚拟通道分时共享

开关枢纽结构形式
...

参考多端口存储器思想，把互连结构的开关设置在处理机和接口内部，组成分布式结构，称为开关枢纽结构形式

每台处理机通过它的开关枢纽和别的多态处理机连接组成各种有分布结构的多处理机

理想情况：

  • 所用开关枢纽少
  • 每个开关枢纽端口不多
  • 能用较短路径连接很多处理机
  • 不改变模块本身结构
    • 系统规模可以任意扩充

存储器的组织
...

一般性的，使用多个模块构成的并行存储器，这种存储器有两种编址策略，在之前的章节中提到过:

  • 低位交叉编址
    • 适用于数据分散，各个单元基本不共享的系统
  • 高位交叉编址
    • 适用于数据有大块的集中的系统

本地存储器
...

每个处理机的全部活跃页面存放在存储器模块 i 中，这种存储器模块 i 又不包含其他处理机的页面，就不会又访存冲突了

而这种本地存储器的概念延申，当存储器模块多于处理机数时，可用若干个存储器模块的集合作为某个处理机的本地存储器

也就是二维的并行处理机构形

多 Cache 的一致性问题
...

进程迁移
...

当某一处理器的进程可能会暂停并迁移到另外一个处理机上 但在进行这样的操作时，可能进程在原先的处理机上进行过 Cache 的数据修改，而迁移后修改就失效了

解决方法：

  • 禁用进程迁移
  • 在进程挂起时，靠硬件方法将 Cache 改写过的信息强制写回主存

硬件为基础实现 Cache 的一致性
...

  • 总线多处理机（监视 Cache 协议）
    • 写作废法
    • 写更新法
  • 多级网络多处理机（目录表法）
    • 全映像目录表
      • 表中每个标志位对应 N 个处理机
    • 有限目录表法
      • 限制了同时使用某块数据的处理机数
    • 链式目录表法
      • 每个 Cache 中存具有这个数据块的下一个 Cache
      • 使用时顺链查找
  • 软件实现
    • 将程序数据分为可以存入 Cache 和不可以
    • 没有商品化
