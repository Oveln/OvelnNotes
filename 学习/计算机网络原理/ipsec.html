IPsec

简介
...

IPsec 作为建立在网络层上的 VPN 隧道协议，有以下体系结构

模式
...

  • 主机模式
    • IPsec 数据报的发送和接受均由端系统完成
    • IPsec 是主机感知的
  • 隧道模式
    • IPsec 数据报由边缘路由器完成
    • 边缘路由器是 IPsec 感知的

服务
...

  • 机密性
  • 数据完整性
    • 通过密码散列函数
  • 源认证/鉴别
  • 重放攻击预防

安全关联（SA）
...

在数据发送前，发送实体和接受实体之间需要建立安全关联

  • SA 是单工的：单向

发送实体和接收实体需要维护 SA 的状态信息

  • IP 是无连接的，但 IPsec 是面向连接的

由于 SA 是单工的，所以一般一个 VPN 的连接中，会有两个 SA

  • A-->B，B-->A

安全关联主要参数
...

  • 安全参数索引（SPI）：32 位 SA 唯一标识
  • 加密密钥、认证密钥
  • 密码算法标识
  • 序列号
    • 防重放攻击
  • 抗重播窗口
  • 生存周期
    • 规定 SA 的有效使用周期
  • 运行模式：传输模式或隧道模式
  • IPSec 隧道源，目的地址

安全关联数据库（SAD）
...

IPsec 端点将 SA 状态保存在安全关联数据库 SAD 中

  • 在处理 IPsec 数据报时，通过 SPI 检索SAD定位这些信息

如：对于 n 个销售人员，1 个分支机构的 VPN，总部的路由器 R1 的 SAD 中存储 2+2n 条 SA

安全策略数据库（SPD）
...

并非所有数据报都经过 IPsec，安全策略（SP）定义了什么样的数据流实施什么样的安全处理

  • 应用 IPsec
  • 绕过 IPsec
  • 丢弃

SP 组成了 SPD，每一条记录就是一条 SP

  • 当收到数据报时，会提取关键信息填充到选择符中
    • 关键信息：目的 IP，源 IP，传输层协议，源，目标端口
    • 利用选择符搜索 SPD，匹配 SP
  • 利用匹配到的 SP 去进行处理该数据报

传输模式 ESP
...

IP 数据报的处理在主机上构造一个新的 IP 数据报，将原 IP 数据报的载荷提取出来，并在原 IP 数据报载荷头尾添加 ESP 头，ESP 尾，ESP 认证数据并加密原 IP 数据报和尾部的内容由认证内容提供 ESP 头部至尾部的认证

隧道模式 ESP
...

IP 数据报的处理在路由器上构造一个新的 IP 数据报，将原 IP 数据报作为新的数据报的载荷并在原 IP 数据报添加 ESP 头、ESP 尾、认证数据并加密原 IP 数据报和 ESP 尾由认证内容提供 ESP 头至 ESP 尾的认证

添加 ESP 尾部时为了更好的实现现代加密技术 > 分组密码

运行流程（隧道模式ESP）
...

发送方路由器：

  • 检索 SPD，决定对该 IP 数据报的处理策略
  • 检索 SAD，确定 SA
  • 在原数据报后附加 ESP 尾部，利用 SA 定义的算法和密钥，加密上述结果
  • 添加 ESP 头部，创建 enchilada
  • 对整个 enchilada，用 SA 定义的算法和密钥，创建报文认证 MAC 并附加在 enchilada 后
  • 构造新 IP 头，上述内容作为 IP 数据报载荷
  • 将新 IP 数据报发送到互联网

接收方路由器：

  • 收到 IP 数据报
  • 从 IP 数据报提取选择夫，检索 SPD，确定处理策略
  • 判断是否为 IPsec
  • 如果时，从 ESP 头部提取 SPI，检索 SAD
    • 如果找不到 SA，触发 IKE 或丢弃
    • 如果找到，根据 SA 解封数据报，得到原始 IP 数据报
  • 将原始数据报发送给主机

IPsec 实现（一般以隧道模式 ESP 说明）
...

IPsec 序列号
...

  • 对于新 SA，发送方初始化序列号 0
  • 每次通过 SA 发送数据报
    • 发送方增加序列号计数器
    • 将计数器值至于序列号字段
  • 目的：
    • 预防嗅探和回访攻击
    • 因为接收重复的已认证的 IP 分组会破坏正常服务
  • 判断是否问题 IP 分组方法：
    • 接收方检验分组重复
    • 无需记录所有分组，而是利用一个窗口滑动窗口协议，与窗口内的序列号不重复，就判断没有问题

SA 建立和密钥管理
...

  • 手工方式
    • 所有信息手动配置
    • SA 永久有效
    • 适用于网络简单的情况
  • 自动方式
    • SA 通过双方的协商方式产生
    • SA 过期后需要重新协商，提高安全性
    • 适用于拓朴结构较为复杂的网络，或安全性要求较高的网络

简介
...

IKE 定义了在 Internet 上协商、建立、修改、删除 SA 的通用框架

使用了 OAKLEY 的密钥交换模式

  • 一个密钥交换协议，允许认证过的双方通过不安全的网络交换密钥参数

使用了 SKEME 的共享和更新密钥技术

  • 提供了 IKE 交换密钥的算法

IKE 为 IPsec 提供关于 SA 建立的服务：

  • 密钥交换与管理
  • 身份认证：通信对等体之间的认证
  • SA 协商管理

认证方式
...

IKE 认证可以通过：

  • 预共享密钥（PSK）、
  • 公钥基础设施（PKI）（公开/私钥对及其证书）

通过 PSK 和PKI 可以认证彼此，并建立 IPsec SA（每个方向一个） PKI 的认证过程类似于 SSL

IKE 的阶段
...

  • 建立双向 IKE SA（也成为 ISAKMP 安全关联）
    • 为双方进一步的 IKE 通信提供机密性、数据完整性以及数据源认证服务
    • 不同于 IPsec SA
    • 两种模式
      • 野蛮模式：进行三个信息交互，使用信息较少
      • 主模式：进行六次信息交互，提供身份保护并且更灵活
        • 前四次消息明文，后两次加密
  • 基于 ISAKMP 协议，进行 IPsec SA 协商
