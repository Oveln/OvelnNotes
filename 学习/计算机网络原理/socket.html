Socket

简介
...

  • 进程间通信利用 socket 发送/接收信息实现
  • 通过（IP+端口号）来定位网络层上的其他进程（对外）
  • 操作系统/进程通过套接字描述符来引用套接字（对内）
  • 类似于寄信 - 发送方将信息送到门外信箱 - 发送方以来（门外的）传输基础设施将消息传到接收方所在主机，并送到接收方的门外 - 接收方从门外获取消息
  • 传输基础设施（操作系统）向进程提供的传输层 API
    • 传输协议的选择
    • 参数的设置

套接字类型
...

  • SOCK_STREAM 流式套接字
    • 面向 TCP
    • 可靠，面向连接，字节流传输，点对点
  • SOCK_DGRAM 数据报类型套接字
    • 面向 UDP
    • 不可靠，无连接，数据包传输
  • SOCK_RAW 原始套接字
    • 面向网络层

循环无连接服务器
...

  1. 创建套接字，绑定端口号）
  2. 反复接收客户端请求
  3. 遵循应用层协议，构造响应报文，发送给客户
  • 不能使用 connect 函数，使用 sendto 发送数据报

循环连接服务器
...

  1. 创建套接字，绑定端口号
  2. 设置为被动监听模式（listen）
  3. 调用 accept 接收下一个连接请求，并创建新套接字
  4. 遵循应用层协议，反复接收请求，并构造响应报文，用新套接字发送给客户
  5. 关闭于该客户的连接，返回第三步

主线程
...

  1. 创建套接字，绑定端口号
  2. 反复调用 recvfrom 函数接收用户请求并创建新进程处理该客户响应

子线程
...

  1. 接受一个特定请求
  2. 根据应用层协议构造响应报文，并发给客户
  3. 推出（一个子线程处理一个请求后立即终止）

并发连接服务器
...

主线程
...

  1. 创建套接字，绑定端口号
  2. 设置为监听模式
  3. 反复调用 accept 函数接收下一个连接请求，并创建新的子线程处理该客户请求

子线程
...

  1. 接收客户的服务请求
  2. 遵循应用层协议与特定用户进行交互
  3. 关闭/释放连接并退出 （进程终止）
