ICG 游戏
...

上面的 Nim 游戏就是一个简单的 ICG 游戏，接下来我们对 ICG 游戏进行一个完整的定义

  1. 游戏有两人参与，两人轮流作出对于自己最优的决策
  2. 当有一人无法决策时，该人失败。且无论如何决策，游戏必然在有限时间内结束
  3. 游戏中的同一个状态不能达到多次，且没有平局

任何一个 ICG 游戏都存在必胜态以及必败态，在上面的 Nim 问题中我们已经略观一二，展开的说就是：

  1. 无法移动的状态为必败态
  2. 可以移动到必败态的状态为必胜态
  3. 所有移动都会进入到必胜态的状态为必败态

对于这样的一个游戏，可以将所有状态看作结点，状态之间的转移关系看作边。通过这个操作，我们能够获得一张有向无环图

有向：由状态转移的有向性保证 无环：在第三条定义中，说明了同一个状态无法到达多次

当然，可以对某个 ICG 游戏建立其状态图，然后进行遍历来知道某个状态是必胜态还是必败态 但这太过耗时，实际上我们可以通过一个函数来计算它，也就是 SG 函数

容易写出 Sg 函数代码

int prime[78498]//预处理的质数数组(一个冷知识，小于10^6的质数数量为78498)
int lazy_sg[1000005];//使用记忆化,初值设为-1
int Sg(int x) {
	if (lazy_sg[x]!=-1) return lazy_sg[x];
	int sqrtx = sqrt(x);
	std::unordered_set<int> S;
	for (int i=1;prime[i]<=sqrtx;i++) {//从第二个质数开始，第一个是2，不需要扫
		if (x%prime[i]!=0) continue;
		S.insert(Sg(x/prime[i]));
	}
	//进行mex运算
	for (int i=1;;i++) {//从1开始，因为任何x都可以转移到Sg=0的情况
		if (!S.count(i)) return lazy_sg[x]=i;
	}
}

int Sg(int x,int p) {
	if (p%2) {
		if (x>p) return Sg(x%(p+1),p);
		if (x==p) return 2;
		return x%2;
	} else {
		return x%2;
	}
}
